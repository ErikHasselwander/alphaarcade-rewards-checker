Search for transactions. Transactions are returned oldest to newest unless the address parameter is used, in which case results are returned newest to oldest.

#### Parameters

Try it out

| Name | Description |
| --- | --- |
| limit<br>integer<br>(query) | Maximum number of results to return. There could be additional pages even if the limit is not reached. |
| next<br>string<br>(query) | The next page of results. Use the next token provided by the previous results. |
| note-prefix<br>string<br>(query) | Specifies a prefix which must be contained in the note field. |
| tx-type<br>string<br>(query) | _Available values_ : pay, keyreg, acfg, axfer, afrz, appl, stpf, hb<br>--paykeyregacfgaxferafrzapplstpfhb |
| sig-type<br>string<br>(query) | SigType filters just results using the specified type of signature:<br>- sig - Standard<br>- msig - MultiSig<br>- lsig - LogicSig<br>_Available values_ : sig, msig, lsig<br>--sigmsiglsig |
| txid<br>string<br>(query) | Lookup the specific transaction by ID. |
| round<br>integer<br>(query) | Include results for the specified round. |
| min-round<br>integer<br>(query) | Include results at or after the specified min-round. |
| max-round<br>integer<br>(query) | Include results at or before the specified max-round. |
| asset-id<br>integer<br>(query) | Asset ID |
| before-time<br>string($date-time)<br>(query) | Include results before the given time. Must be an RFC 3339 formatted string. |
| after-time<br>string($date-time)<br>(query) | Include results after the given time. Must be an RFC 3339 formatted string. |
| currency-greater-than<br>integer<br>(query) | Results should have an amount greater than this value. MicroAlgos are the default currency unless an asset-id is provided, in which case the asset will be used. |
| currency-less-than<br>integer<br>(query) | Results should have an amount less than this value. MicroAlgos are the default currency unless an asset-id is provided, in which case the asset will be used. |
| address<br>string<br>(query) | Only include transactions with this address in one of the transaction fields. |
| address-role<br>string<br>(query) | Combine with the address parameter to define what type of address to search for.<br>_Available values_ : sender, receiver, freeze-target<br>--senderreceiverfreeze-target |
| exclude-close-to<br>boolean<br>(query) | Combine with address and address-role parameters to define what type of address to search for. The close to fields are normally treated as a receiver, if you would like to exclude them set this parameter to true.<br>--truefalse |
| rekey-to<br>boolean<br>(query) | Include results which include the rekey-to field.<br>--truefalse |
| application-id<br>integer<br>(query) | Application ID |
